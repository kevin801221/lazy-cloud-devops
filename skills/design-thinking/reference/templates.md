# Design Thinking — 可操作模板與腳本

複製即用。每個模板對應 SKILL.md 的一個階段。

---

## 1. Empathy Map（同理心地圖）— Empathize

對著「一位具體使用者」填，不要填「平均使用者」。

```
┌─────────────────────────┬─────────────────────────┐
│ SAYS（他說的原話）       │ THINKS（他可能在想的）   │
│ - "……"                  │ - 真正在意但沒說出口的   │
│ - "……"                  │ - 猶豫、疑慮             │
├─────────────────────────┼─────────────────────────┤
│ DOES（他實際做的行為）   │ FEELS（他的情緒）        │
│ - 觀察到的動作           │ - 沮喪 / 焦慮 / 興奮      │
│ - 與 SAYS 的落差！       │ - 什麼觸發了情緒         │
└─────────────────────────┴─────────────────────────┘
        PAINS（痛點）          │      GAINS（渴望）
   - 阻礙、風險、挫折          │ - 想達成的、想要的好處
```

> 重點看 **SAYS vs DOES 的落差**——人說的和做的不一致處，藏著最真的洞察。

## 2. 使用者訪談腳本 — Empathize

**開場**（卸下防備）
- 「沒有對錯答案，我是來學習的，不是來考你的。」
- 「可以盡量說細節，包括你覺得很蠢的部分。」

**核心問題**（問過去具體事件，不問未來假設）
- 「可以跟我說說**上一次**你 ____ 的情況嗎？從頭到尾。」
- 「那時候**最讓你卡住**的是什麼？」
- 「你後來**怎麼解決**的？有沒有用什麼土法煉鋼的辦法（workaround）？」
- 「如果有一根魔杖，你會希望那一刻**什麼不一樣**？」
- 連續追問：「為什麼？」×5

**禁止**
- ❌「你會不會用一個能 ____ 的 app？」（假設性 + 引導）
- ❌「你覺得這個點子好不好？」（討讚）

## 3. POV 陳述模板 — Define

```
[使用者，要具體] 需要 [動詞開頭的需求]，
因為 [出乎意料的洞察 — 不能是廢話]。
```

**自檢**
- 使用者夠具體嗎？（「上班族」< 「沒時間開伙的雙薪家庭父母」）
- 需求是「需求」還是偷藏的「解法」？（❌「需要一個外送 app」）
- 洞察讓人「喔！」一聲了嗎？還是只是常識？

---

## 4. How Might We（HMW）模板 — Define

```
我們可以怎麼樣（How might we）________________________？
```

**從一個 POV 衍生多個 HMW**，調整切入角度：
- 放大好處：HMW 讓 ____ 變成一種享受？
- 移除痛點：HMW 讓 ____ 完全消失？
- 換隱喻：HMW 把 ____ 變得像 ____ 一樣？
- 挑戰假設：HMW 根本不需要 ____ 就能 ____？

**甜蜜點檢查**：太窄（已含解法）❌ ｜太寬（無從下手）❌ ｜剛好（打開空間又有方向）✅

## 5. 發想引導腳本 — Ideate

**規則（貼牆上）**
1. 暫緩評judgment — 不准說「但是」「行不通」。
2. 鼓勵瘋狂點子。
3. 疊加別人的（「Yes, and...」）。
4. 扣題（對著 HMW）。
5. 一次一個人講。
6. **衝量** — 目標數字寫出來（如 30 個 / 10 分鐘）。

**收斂步驟**
1. 沉默整理：相似點子分群。
2. 點點投票：每人 3 票。
3. 2×2（影響力 × 可行性），挑右上。
4. 選 2-3 個做原型（不要只留 1 個）。

---

## 6. 原型規劃 — Prototype

填這張表再動手，避免做過頭：

```
我這個原型最想回答的「一個」問題：____________________
能回答它的「最低保真度」是：  紙稿 / 線框 / 可點 / 綠野仙蹤
預計花費時間（越短越好）：     ____ 小時
做完要拿給誰測：               ____（真實使用者，非同事）
我願意把它丟掉嗎？             是 ✅ / 否（→保真度做太高了）
```

---

## 7. 使用者測試腳本 — Test

**開場**
- 「測的是這個設計，不是你。卡住完全是設計的錯。」
- 「請邊操作邊把你心裡想的講出來。」

**進行中**
- 給任務，不給步驟：「假設你想 ____，請開始。」
- **閉嘴**。使用者沉默時忍住，不要救他。
- 卡住時只問：「你現在在想什麼？」「你預期點下去會怎樣？」

**禁止**
- ❌ 替他操作、解釋介面、暗示「應該點那裡」。
- ❌ 問「好不好用？」（他會禮貌說好）。

**記錄表**
```
| 任務 | 卡住點 | 誤解了什麼 | 驚喜/皺眉 | 原話 |
|------|--------|-----------|----------|------|
```

**測完**：5 位使用者即可抓到約 80% 主要問題 → 整理發現 → 回到對應階段迭代。
